# JCImpact ‚Äî Dashboard Update Workflow

## Overview

The dashboard reads from a single file: `src/data/data.json`.
That file is generated by `scripts/update-data.py` from your Excel workbook.
**No code changes are ever needed to update the dashboard data.**

---

## Every Update Cycle (3 steps)

```bash
# Step 1 ‚Äî Drop new Excel on Desktop (keep the same filename), or note the path.

# Step 2 ‚Äî From the project root, run:
npm run update-data

# Or if your file is named differently or in another location:
python3 scripts/update-data.py ~/Downloads/JCIMPACT_April.xlsx
# Or via env var:
JCIMPACT_EXCEL=~/Downloads/JCIMPACT_April.xlsx npm run update-data

# Step 3 ‚Äî Refresh localhost:3000 in your browser.
```

That's it. The script prints a summary report and flags any data gaps automatically.

## Required Excel File Structure

The script expects these **exact sheet names** (case-sensitive):

| Sheet Name | What the script reads |
|---|---|
| `CITYWIDE` | Monthly crime totals (7 Major Impact Categories) ¬∑ period dates ¬∑ last-updated date |
| `NORTH` | District monthly crime breakdown |
| `EAST` | District monthly crime breakdown |
| `SOUTH` | District monthly crime breakdown |
| `WEST` | District monthly crime breakdown |
| `MVA Data` | Individual MVA crash records |
| `Shots Fired & Shootings` | Individual shooting/shots-fired incidents |

Optional (parsed if present, ignored if missing):
- `MVA Summary` ‚Äî not currently read; district MVA counts come from `MVA Data` directly.
- `YTD Summary`, `Supplemental` ‚Äî not currently read.

## Required Column Names Per Sheet

### CITYWIDE / NORTH / EAST / SOUTH / WEST sheets

The script detects these automatically by **row layout**, not column letters.
Each monthly block must follow this structure:

```
Row N:   ‚ñå  JANUARY 2026   (01/01 ‚Äì 01/31)          ‚Üê section header
Row N+1: OFFENSE | Note | Wk1 | Wk2 | ... | JAN Total | vs Prev Mo | vs 2025 %
Row N+2: ‚òÖ 7 MAJOR IMPACT CATEGORIES
Row N+3: Assault Total   ...weekly values...   JAN Total
Row N+4:   Aggravated Assault   (indented subcategory ‚Äî NOT counted in total)
Row N+5: Stolen Vehicles
...
```

**The 7 Major Impact Categories counted toward `totalCrimes`:**

| Row label (exact) | Notes |
|---|---|
| `Assault Total` | Parent row; subcategories (indented) are excluded |
| `Stolen Vehicles` | |
| `Burglary Total` | Parent row only |
| `Homicide` | Also used for the `homicides` stat |
| `Theft` | Excludes Theft of Auto Parts |
| `Sex Offenses` | |
| `Robbery Total` | Parent row only |

**Not counted:** `Theft of Auto Parts`, `Bias Incidents`, `Drug Arrests`, `Shoplifting`
**Not counted:** Any row with leading whitespace (indented subcategories)

The script reads whichever is higher: **sum of weekly columns** or **value in the `MON Total` column** ‚Äî so either data-entry style works.

### MVA Data sheet

| Required column | Description |
|---|---|
| `Case #` | Case number like `26-040062`; used as incident ID and year detection |
| `Date` | Format `MM/DD` (year inferred from case number prefix) |
| `Road` | Primary road name |
| `Cross Street` | Cross street (optional; blank = point location) |
| `District` | `NORTH`, `EAST`, `SOUTH`, or `WEST` (case-insensitive) |

The script finds these columns by name ‚Äî column order does not matter.

### Shots Fired & Shootings sheet

The script detects two sections by header pattern:

```
SHOTS FIRED (8 YTD  |  JAN=3, FEB=1, MAR=4)   ‚Üê section trigger
Date | Type | Location | Arrest                 ‚Üê column headers
1/12/26 | Shots Fired | Cator & Ocean | Yes     ‚Üê data rows

SHOOTING HITS (5 YTD  |  JAN=2, FEB=2, MAR=1)  ‚Üê section trigger
Date | Type | Location | Arrest
1/1/26  | Shooting Hit | 352 West Side Ave | No
```

| Column | Required | Notes |
|---|---|---|
| `Date` | ‚úÖ | Format `M/D/YY` or `M/D/YYYY` |
| `Type` | ‚úÖ | `Shots Fired` or `Shooting Hit` |
| `Location` | ‚úÖ | Address or intersection; used for geocoding |
| `Arrest` | ‚úÖ | `Yes` / `No` |
| `District` | ‚öÝÔ∏è **Recommended** | `North` / `East` / `South` / `West` ‚Äî see Known Data Gaps below |

## Geocode Cache

- File: `scripts/geocode-cache.json`
- **First run:** geocodes all addresses (~1‚Äì2 min at 12 concurrent Mapbox API requests)
- **Subsequent runs:** only NEW or changed addresses are geocoded (seconds)
- The cache is committed to git ‚Äî persists across machines and teammates
- Cache key = the geocode query string (road + cross-street or intersection name)
- Fallback for failed geocodes: deterministic jitter around the district centroid


All dashboard components ‚Äî stat cards, district table, rankings, monthly chart, and the map ‚Äî read exclusively from this file.

## Known Data Gaps

### 1 ‚Äî Assault Total rows not populated for partial months

**What happens:** When a month is still in progress, the `Assault Total` parent row in the district sheets is sometimes left blank (weekly data entered only on the `Aggravated Assault` sub-row). Because `Assault Total` is a Major Impact Category, this causes the month's crime count to be underreported.

**When this affects you:** Any time you run the update mid-month before your weekly CompStat entry is finalized.

**The warning you'll see:**
```
‚öÝ  DATA GAP DETECTED: 73 crimes appear in CITYWIDE but not district sheets.
   Most likely cause: "Assault Total" rows in one or more district sheets
   for the current partial month are blank ‚Äî only subcategories were entered.
   Fix: fill in the "Assault Total" row in NORTH/EAST/SOUTH/WEST sheets.
```

**The fix:** Fill in `Assault Total` in **row 49, column H** (`H49`) of each district sheet with the correct MAR/APR/etc. Total value. Weekly cells (C49‚ÄìG49) can be filled in too for completeness, but the script will use whichever value is higher.

**Current March 2026 state (as of last update 03/19/2026):**

> The data only covers through 03/21, so Wk4 (3/22‚Äì3/28) and Wk5 data do not exist yet.
> But even Wks 1‚Äì3 were not entered in `Assault Total` (row 49) on any district sheet.
> Only the sub-row `Aggravated Assault` (row 50) has weekly data.

Cells that need values entered (once you have the numbers):

| Sheet | Wk1 (C49) | Wk2 (D49) | Wk3 (E49) | MAR Total (H49) |
|---|---|---|---|---|
| NORTH | blank | blank | blank | formula = 0 |
| EAST | blank | blank | blank | formula = 0 |
| SOUTH | blank | blank | blank | formula = 0 |
| WEST | blank | blank | blank | formula = 0 |

Minimum known values (Aggravated Assault alone, Wks 1‚Äì3 only ‚Äî Assault Total will be higher):

| Sheet | Agg Assault Wk1 | Wk2 | Wk3 | Partial floor |
|---|---|---|---|---|
| NORTH | 5 | 3 | 5 | ‚â• 13 |
| EAST | 4 | 4 | 2 | ‚â• 10 |
| SOUTH | 5 | 7 | 1 | ‚â• 13 |
| WEST | 2 | 3 | 6 | ‚â• 11 |

### 2 ‚Äî No District column in the Shots Fired & Shootings sheet

**What happens:** Without a `District` column, the script assigns shooting incidents to districts by finding the nearest district centroid after geocoding. This is ~80% accurate but can misassign incidents near district boundaries.

**The fix (planned next):** Add a `District` column to both sections of the `Shots Fired & Shootings` sheet with values `North` / `East` / `South` / `West`. The script already looks for this column ‚Äî it just isn't present yet.

**The tip you'll see when this column is missing:**
```
TIP: Add a "District" column to the "Shots Fired & Shootings" sheet
     for accurate per-district shooting counts. Without it, district
     assignment is estimated from geocoded coordinates.
```

### 3 ‚Äî MVA data covers March only (NJ Crash Reports lag)

**What happens:** The NJ Crash Statistical Reports for January and February 2026 were not yet available when the `MVA Data` sheet was built. All 333 MVAs are dated in March, causing the monthly trend chart to show 0 MVAs in January and February and a spike in March.

**How the dashboard handles it:** A footnote on the Monthly Trends chart reads: *"‚ÄÝ MVA data: NJ Crash Reports ‚Äî Jan‚ÄìFeb not yet available"*

**The fix:** When NJ Crash data for Jan‚ÄìFeb becomes available, add those records to the `MVA Data` sheet with their correct dates. Re-run `npm run update-data` ‚Äî no other changes needed.

## Year Transition (2026 ‚Üí 2027)

When you start a new year's workbook:
- MVA case numbers like `27-001234` are auto-detected as 2027
- The script infers the year from the first two digits of case numbers
- Monthly section headers (`‚ñå  JANUARY 2027`) are parsed automatically
- The period string in `meta` updates to `YTD 2027 (Jan '27 ‚Äì ...)`
- No code changes needed
